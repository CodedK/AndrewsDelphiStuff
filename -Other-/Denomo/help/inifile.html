Denomo supports a local or global denomo.ini file to control Denomo's behavior. So now you usually don't need to touch the source code and thus no any version conflict can occur.

Denomo searches for denomo.ini in the following sequence:
1, The directory from which the application is loaded.
2, The current directory.
3, The root directory of currect logical disk from which the application loaded.
4, {CSIDL_LOCAL_APPDATA}\denomo. Usually it's C:\Documents and Settings\username\Local Settings\Application Data\denomo
5, {CSIDL_APPDATA}\denomo. Usually it's C:\Documents and Settings\username\Application Data\denomo
6, {CSIDL_COMMON_APPDATA}\denomo. Usually it's C:\Documents and Settings\All Users\Application Data\denomo
7, The Windows home directory. Usually it's C:\WINNT or C:\Windows

Create a new denomo.ini:
To create a new denomo.ini, you only need to create an empty file named "denomo.ini" in the directories mentioned above. Then run the program compiled with Denomo once, Denomo will fill the .ini file fields with the default values and then you can modify the values.

The structure of denomo.ini
There is no mystery in the .ini file. Each field in it is just the variable name of source file DenomoConfig.pas, and the default value is same as the variable value.
